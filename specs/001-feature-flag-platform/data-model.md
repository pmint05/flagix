## Platform Entities

### Organization

Root tenant entity. All resources belong to exactly one organization.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| id | uuid | PK, auto-generated | Primary key |
| name | varchar(255) | NOT NULL, UNIQUE | Organization display name |
| slug | varchar(100) | NOT NULL, UNIQUE | URL-safe identifier |
| createdAt | timestamp | NOT NULL, default now() | Creation timestamp |
| updatedAt | timestamp | NOT NULL, default now() | Last update timestamp |

**Relationships**:
- Has many Projects (1:N)
- Has many OrganizationMembers (1:N)

**Validation Rules**:
- name: 1-255 characters, non-empty
- slug: 1-100 characters, lowercase alphanumeric + hyphens, auto-generated from name

---

### OrganizationMember

Junction table linking Better Auth users to organizations with role assignments. A user can belong to multiple organizations with different roles.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| id | uuid | PK, auto-generated | Primary key |
| userId | string | FK → user.id (Better Auth), NOT NULL | References Better Auth user table |
| organizationId | uuid | FK → Organization.id, NOT NULL | Parent organization |
| role | enum('admin', 'editor', 'viewer') | NOT NULL, default 'viewer' | User's role within this organization |
| createdAt | timestamp | NOT NULL, default now() | Membership creation timestamp |
| updatedAt | timestamp | NOT NULL, default now() | Last update timestamp |

**Constraints**:
- UNIQUE(userId, organizationId) — one membership per user per organization
- FK userId references Better Auth `user.id` (string type, not uuid)

**Relationships**:
- Belongs to User (Better Auth) (N:1)
- Belongs to Organization (N:1)

**Validation Rules**:
- userId: must reference an existing Better Auth user
- organizationId: must reference an existing organization
- role: must be one of 'admin', 'editor', 'viewer'

### FeatureFlag

Core entity controlling feature availability. Scoped to a project and environment.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| id | uuid | PK, auto-generated | Primary key |
| environmentId | uuid | FK → Environment.id, NOT NULL | Parent environment |
| key | varchar(255) | NOT NULL | Unique flag identifier |
| name | varchar(255) | NOT NULL | Display name |
| description | text | NULLABLE | Optional description |
| flagType | enum('boolean', 'multivariate') | NOT NULL, default 'boolean' | Flag mode |
| status | enum('draft', 'active', 'archived') | NOT NULL, default 'draft' | Lifecycle state |
| isEnabled | boolean | NOT NULL, default false | Global kill switch |
| defaultVariationId | uuid | FK → Variation.id, NULLABLE | Default variation (set after creation) |
| createdAt | timestamp | NOT NULL, default now() | Creation timestamp |
| updatedAt | timestamp | NOT NULL, default now() | Last update timestamp |
| version | integer | NOT NULL, default 1 | Optimistic locking version |

**Constraints**:
- UNIQUE(environmentId, key) — flag key unique within environment
- CHECK: defaultVariationId must reference a variation belonging to this flag

**Relationships**:
- Belongs to Environment (N:1)
- Has many Variations (1:N)
- Has many TargetingRules (1:N)
- Has many AuditLogs (1:N, via entityId where entityType = 'feature_flag')
- References defaultVariation → Variation (1:1)

**State Transitions**:
```text
draft → active    (user activates flag)
active → archived (user archives flag)
```
- No backward transitions allowed (FR-011)
- Only `active` flags are evaluated (FR-009)
- `draft` and `archived` flags return safe default (FR-008, FR-010)

**Validation Rules**:
- key: 1-255 characters, alphanumeric + hyphens + underscores, unique per environment
- name: 1-255 characters
- flagType: must be 'boolean' or 'multivariate'
- status: must follow valid state transitions
- defaultVariationId: must reference a variation of this flag (if set)

### Variation

Possible output value of a feature flag.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| id | uuid | PK, auto-generated | Primary key |
| featureFlagId | uuid | FK → FeatureFlag.id, NOT NULL | Parent flag |
| key | varchar(100) | NOT NULL | Variation identifier |
| value | jsonb | NOT NULL | Resolved output value (boolean, string, or JSON object) |
| description | text | NULLABLE | Optional description |
| createdAt | timestamp | NOT NULL, default now() | Creation timestamp |
| updatedAt | timestamp | NOT NULL, default now() | Last update timestamp |

**Constraints**:
- UNIQUE(featureFlagId, key) — variation key unique within flag

**Relationships**:
- Belongs to FeatureFlag (N:1)
- Referenced by FeatureFlag.defaultVariationId (1:1)
- Referenced by TargetingRule.variationId (1:N)

**Validation Rules**:
- key: 1-100 characters, alphanumeric + hyphens + underscores
- value: must match flag's flagType:
  - BOOLEAN flag: value must be boolean (true/false)
  - MULTIVARIATE flag: value can be string or JSON object
- BOOLEAN flags auto-create "true" and "false" variations on creation (FR-018)

### TargetingRule

Defines targeting logic for a feature flag.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| id | uuid | PK, auto-generated | Primary key |
| featureFlagId | uuid | FK → FeatureFlag.id, NOT NULL | Parent flag |
| ruleType | enum('kill_switch', 'user', 'role', 'percentage') | NOT NULL | Rule type |
| priority | integer | NOT NULL | Evaluation order (lower = higher priority) |
| variationId | uuid | FK → Variation.id, NOT NULL | Variation to return when matched |
| conditions | jsonb | NOT NULL | Rule-specific conditions |
| isEnabled | boolean | NOT NULL, default true | Rule-level toggle |
| createdAt | timestamp | NOT NULL, default now() | Creation timestamp |
| updatedAt | timestamp | NOT NULL, default now() | Last update timestamp |

**Constraints**:
- UNIQUE(featureFlagId, priority) — priority unique within flag
- CHECK: variationId must reference a variation belonging to the same flag

**Priority Mapping** (derived from ruleType):
| ruleType | Priority Range | Notes |
|----------|---------------|-------|
| kill_switch | 0 | Highest priority, only one per flag |
| user | 100-199 | User-specific targeting |
| role | 200-299 | Role-based targeting |
| percentage | 300-399 | Percentage rollout |

**Conditions Schema** (by ruleType):

**kill_switch**:
```json
{}
```
No conditions needed — kill switch is a binary on/off toggle.

**user**:
```json
{
  "userIds": ["user-123", "user-456"]
}
```
List of user IDs to match against evaluation context userId.

**role**:
```json
{
  "roles": ["beta-testers", "internal"]
}
```
List of roles to match against evaluation context role.

**percentage**:
```json
{
  "percentage": 30
}
```
Integer 0-100 representing the percentage of users to include.

**Relationships**:
- Belongs to FeatureFlag (N:1)
- References Variation (N:1)

**Validation Rules**:
- ruleType: must be one of the four enum values
- priority: must be within the valid range for the ruleType
- variationId: must reference an existing variation of the parent flag
- conditions: must match the schema for the given ruleType
- Only one kill_switch rule per flag

### AuditLog

Immutable record of all state-mutating operations.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| id | uuid | PK, auto-generated | Primary key |
| organizationId | uuid | FK → Organization.id, NOT NULL | Tenant scope |
| projectId | uuid | FK → Project.id, NULLABLE | Project scope (if applicable) |
| actionType | enum('create', 'update', 'delete', 'toggle') | NOT NULL | Type of mutation |
| entityType | enum('organization', 'project', 'environment', 'feature_flag', 'targeting_rule', 'variation') | NOT NULL | Type of entity mutated |
| entityId | uuid | NOT NULL | ID of the mutated entity |
| actorId | string | FK → user.id (Better Auth), NULLABLE | Authenticated user who performed the action (null for system actions) |
| actorType | enum('user', 'system') | NOT NULL, default 'user' | Actor classification |
| actorEmail | varchar(255) | NULLABLE | User email at time of action (snapshot for historical accuracy) |
| changes | jsonb | NOT NULL | Before/after state diff |
| timestamp | timestamp | NOT NULL, default now() | When the mutation occurred |

**Constraints**:
- No UPDATE or DELETE operations allowed (immutable)
- FK actorId references Better Auth `user.id` (string type); NULLABLE to support system-initiated actions
- Index on (organizationId, timestamp) for efficient querying
- Index on (entityId, entityType) for entity-specific queries
- Index on (projectId, timestamp) for project-scoped queries
- Index on (actorId) for user-specific audit queries

**Relationships**:
- Belongs to Organization (N:1)
- Optionally belongs to Project (N:1)
- References Better Auth User (N:1) via actorId (when actorType = 'user')
- References mutated entity via entityId + entityType

**Changes Schema**:
```json
{
  "before": { ... },
  "after": { ... }
}
```
- `before`: null for create operations
- `after`: null for delete operations
- Both populated for update and toggle operations

**Retention Policy** (FR-063, FR-064, FR-065):
- Active storage: 90 days from creation
- After 90 days: archived to cold storage
- Archived logs remain queryable with higher latency
